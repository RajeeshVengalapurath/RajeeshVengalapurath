Raptoroid - User Management

Module


Multiple User

The software supports multiple user logins. Means (say) Adam and Daniel can have separate user logins.

Advantages

Admin can set up accounts with different privileges for each user, and keep an eye on how theyâ€™re using the Software. Each person sees their own permitted modules and favourites when they sign in. Each user's activities are kept separate, nobody else can see it other than the Admin. Multiple people can be signed in at once from different computers.

User Roles

A user role defines permissions for users to perform a group of tasks. A role must be assigned to every user. There are some predefined roles with a predefined set of permissions already there in the software. The fuctionality of these 3 roles cannot be changed by anyone.

The predefined roles are

  • SuperAdmin
  • Admin
  • User

Role: SuperAdmin

  • There can be multiple users with SuperAdmin privilege.
  • This role is assigned only to the Marketer and developer of the software.
  • They can see activities of SuperAdmin, Admin, User and CustomRoles users.
  • They can enable disable features of the software.
  • They can create and manage users belong to SuperAdmin, Admin, User and CustomRoles.
  • The functionality of this role cannot be changed by anyone.

Role: Admin

  • There can be multiple users with Admin privilege.
  • This role is assigned to the administrative users of the client.
  • They can see activities of Admin, User and CustomRoles users.
  • They cannot enable disable features of the software.
  • They can create and manage users belong to Admin, User and CustomRoles.
  • The functionality of this role cannot be changed by anyone.

Role: User

  • There can be multiple users with "User" privilege.
  • This role is assigned to the normal users of the client.
  • They can see activities of them only.
  • They cannot enable disable features of the software.
  • They cannot create and manage any type of users.
  • The functionality of this role cannot be changed by anyone.

Role: Custom Role (eg. Accountant)

  • There can be multiple users with Custom Role privilege.
  • This role is assigned to the normal users of the client.
  • They can see activities of them only.
  • They cannot enable disable features of the software.
  • They cannot create and manage any type of users.
  • The functionality of this role can be changed by SuperAdmin & Admin.

SuperAdmin & Admin can enable/disable the following permissions to modules assigned to "custom role"
